### 设计原则：
 - 单一职责原则
 - 接口隔离原则
 - 依赖倒转原则
 - 里氏替换原则
 - 开闭原则 ocp
 - 迪米特法则
 - 合成服用原则
 
### 设计模式目的：
 - 重用性（相同功能的代码，不用多次编写）
 - 可读性（编程的规范性，便于代码的阅读和理解）
 - 拓展性（方便新功能的增加）
 - 可靠性（新增功能对旧功能不造城影响）
 - 高内聚、低耦合
